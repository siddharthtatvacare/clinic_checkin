Flow A — PolicyHolder PreBooked

Policy Holder Pre-Booked

Policy holder who pre-booked a doctor consult, lab visit, or AHC. Arrives at the kiosk (or uses geo-fenced app check-in). The simplest flow — identify, view orders, check in.

S1Welcome Screen
Taps "Pre-booked appointment"
S2User Identification
Two options: enter mobile number OR scan QR code
Branch: Mobile
S2aMobile + OTP
Enter number → OTP verification → fetch patient + today's orders
OTP Service → GetPatientProfile → FetchOrders
Branch: QR Scan
S2bQR Scan
Scan appointment QR from Visit app → fetch patient + orders
QR Scanner → GetPatientProfile → FetchOrders
S3List of Appointments / Orders
"Welcome {name}" — shows today's appointments (doctor/lab icons). Single "Check-in" CTA for overall clinic check-in.
S4Check-in Confirmation + Token
Token number displayed. WhatsApp sent with token + queue link. Directions to service area (e.g., "Head to Cabin A for Dr. consultation").
QueueService → generateToken

Edge Cases

Patient not found / wrong QR → error message, suggest concierge
OTP wrong 3x → lock out, direct to concierge
Policy holder booked for multiple beneficiaries → S3 lists by beneficiary name, S4 offers loop to check in others

Flow B — Beneficiary PreBooked

Beneficiary Pre-Booked

Beneficiary whose policy holder booked an appointment for them via the Visit app. Arrives with a WhatsApp QR confirmation. Identical structure to Flow A but addressed by beneficiary name.

S1Welcome Screen
Taps "Pre-booked appointment"
S2User Identification
Enter mobile number OR scan QR from WhatsApp
Branch: Mobile
S2aMobile + OTP
Enter number → OTP → fetch patient + today's orders
Branch: QR Scan
S2bQR Scan
Scan appointment QR → fetch patient + orders
S3List of Appointments / Orders
"Welcome {beneficiary_name}" — today's appointments. Single "Check-in" CTA.
S4Check-in Confirmation + Token
Token number, WhatsApp confirmation, directions to service area.

Edge Cases

Patient not found / wrong QR → error, suggest concierge
OTP wrong 3x → lock out, concierge
Multiple beneficiaries pre-booked → S3 lists by name, S4 loop for others

Flow C — PolicyHolder Walk-in

Policy Holder Walk-in

Policy holder walking in without a booking. Identifies via mobile+OTP, selects a beneficiary, picks a service (doctor, lab, or AHC), completes payment, and checks in. Can loop via "Book Another Service" back to beneficiary selection.

S1Welcome
Taps "Walking in"
S2Mobile + OTP
Enter mobile → OTP → fetch patient + beneficiaries
OTP → GetBeneficiaries
S3Select Beneficiary
List of all beneficiaries (including self). Tap one to book for.
S4Select Service
Doctor Appointment · Lab Tests · Available AHCs
FetchServices
S5aDoctor Listing
Available doctors with profiles
FetchDoctorListing
S5bSlot Selection
Available time slots for selected doctor
FetchDoctorSlots
S5cPayment Confirmation
Fee with benefit/wallet deduction breakdown
FetchBenefitEligibility
S5dQR Payment
Scan QR to pay (or auto-proceed if fully covered)
S5eConfirmation + Token
Token, wait time, WhatsApp confirmation
↩ "Book Another Service" → back to S3
S6aLab Test Catalog
Browse by category, "Add" to cart, floating cart icon
FetchLabCatalog
S6bCart Review + Prep Instructions
All selected tests, prices, fasting/prep notes. Remove items.
S6cPayment + Benefits
Benefit breakdown: original, covered, payable
FetchBenefitEligibility
S6dQR Payment
Scan QR (or auto-proceed if zero payable)
S6eLab Confirmation + Token
Test list, lab queue token, directions to Sample Collection
↩ "Book Another Service" → back to S3
S7aAHC Package Listing
Available packages with inclusions, price
FetchAHCPackages
S7bPackage Details + Add-ons
Full test breakdown, prep instructions. "Add more tests" from lab catalog.
FetchLabCatalog
S7cCart Review (Package + Add-ons)
Consolidated cart. Can remove add-ons (not package items).
S7dPayment + Benefits
Package + add-on benefit breakdown
FetchBenefitEligibility
S7eQR Payment
Scan QR (or auto-proceed if zero payable)
S7fAHC Confirmation + Visit Guide
Step-by-step visit flow: Sample Collection → Radiology → Doctor cabin
↩ "Book Another Service" → back to S3

Edge Cases (16 total across 5 categories)

New patient → direct to concierge / Flow E
Zero services for beneficiary → empty state + concierge
Benefit exhausted mid-session → re-call FetchBenefitEligibility at payment
Duplicate test order from EMR → warn at cart review
AHC already done this year → flag on S7a
Doctor slot race condition → return to S5b
QR expiry → "Regenerate QR" option
Session timeout 60s → countdown → reset to S1

Flow D — Beneficiary Walk-in

Beneficiary Walk-in

Beneficiary walking in alone without a booking. Two ID paths: own mobile (auto-confirms identity) or policy holder's mobile (picks from list). Books for self only. "Book Another Service" loops to S4 (not S3).

S1Welcome
Taps "Walking in"
S2Identification — Two Options
(a) Enter own mobile · (b) Enter policy holder's mobile
Path A: Own Mobile
S2aOwn Mobile + OTP
OTP to own phone → check if beneficiary
GetBeneficiaryDetails
S3aBeneficiary Confirmed
"You are a beneficiary under [PH Name]'s policy." Confirm → S4
Path B: Policy Holder's Mobile
S2bPH Mobile + OTP
OTP sent to PH's phone. Beneficiary must coordinate.
GetBeneficiaries
S3bSelf-select from List
Tap own name from beneficiary list → S4
S4Select Service
Doctor · Lab Tests · Available AHCs
FetchServices
S5–S7 (Doctor / Lab / AHC) — identical to Flow C. "Book Another Service" loops to S4 (not S3) since only one beneficiary is in scope.

Unique Edge Cases

S2a: number belongs to policy holder → redirect to Flow C
S2a: number not found → suggest trying PH's number or concierge
S2b: PH's number not found → verify or concierge
S2b: OTP goes to PH's phone → beneficiary must coordinate
S3b: name not in list → "My name is not here" → concierge
No active policy → empty state + concierge

Flow E — Non-Policy Walk-in

Non-Policy Walk-in

Retail or completely new user with no corporate/insurance policy. If unregistered, goes through registration (Visit account + TatvaPractice UHID). Only Doctor and Lab available — no AHC. All payments out-of-pocket.

S1Welcome
Taps "Walking in"
S2Mobile + OTP
Enter mobile → OTP → look up in Visit
GetPatientProfile
If not found → Registration
S2aRegistration Form
Name (req), DOB (req), Gender (req), Email (opt). Mobile pre-filled.
S2bAccount Created
"Welcome to Visit Clinic, [Name]." Visit + TP records created simultaneously.
CreatePatientAccount + CreatePatientRecord (UHID)
S3Select Service
Doctor Appointment · Lab Tests only. No AHC.
FetchServices
S4aDoctor ListingFetchDoctorListing
S4bSlot SelectionFetchDoctorSlots
S4cPayment Confirmation (Full OOP)
No benefit breakdown — full fee shown
S4dQR Payment
S4eConfirmation + Token
↩ "Book Another Service" → back to S3
S5aLab Test Catalog (Add to Cart)FetchLabCatalog
S5bCart Review + Prep Instructions
S5cPayment Confirmation (Full OOP)
No benefit check — full price
S5dQR Payment
S5eLab Confirmation + Token
↩ "Book Another Service" → back to S3

Unique Edge Cases

Duplicate registration race condition → load existing profile, skip reg
Missing required fields → inline validation on S2a
TP record creation fails → error + concierge. ⚠️ DECISION: rollback both or keep Visit account?
No Visit app → WhatsApp still sent to mobile (not app-only)
Session timeout after registration → account persists, only cart discarded
